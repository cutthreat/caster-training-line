# VibeUs Relay — Design Read

Design Read: VibeUs is not another AI community feed. It is a relay station where one person’s finished attempt becomes another person’s starting block.

- Page type: acquisition landing for a public catalogue and community.
- Primary task: open a real public project and understand why contributing back matters.
- Main CTA: `Взять готовое` -> `https://vibeus.app/catalog`.
- Commercial role: orient, prove, then convert.
- DESIGN_VARIANCE: 8/10.
- MOTION_INTENSITY: 4/10.
- VISUAL_DENSITY: 7/10.

## Signature moves

1. Proof relay: real public catalogue items are composed as a handoff chain, not a generic card grid.
2. Visible transfer language: `ВЗЯЛ -> УЛУЧШИЛ -> PASS IT ON` turns the collective-intelligence thesis into structure.
3. Tabbed proof is decision-bearing: each choice opens a real VibeUs project and changes the exact inherited starting point.
4. Dense editorial rhythm alternates with hard black and signal-red fields; the site does not reuse the earlier cinematic or Taste layouts.

## Anti-template boundary

Taste guidance was used only as a failure filter: no AI-purple, no three-equal-benefit cards, no invented metrics, no decorative generated metaphor, no fake loading, no interaction without a decision consequence. The composition, copy, proof chain, colour logic and section rhythm are project-specific.
