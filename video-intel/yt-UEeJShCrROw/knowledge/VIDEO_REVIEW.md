## Useful patterns for Caster

### 1. Input quality is the real leverage

The strongest transferable lesson is the input-first approach. The author explicitly frames poor inputs as the reason outputs become weak or unreachable.

Caster adaptation: every cinematic site needs a production input pack before build: core image, element board, reference roles, scene/section plan, rights, and negative constraints.

Anton risk: beautiful generated media can still misrepresent the product or hide unclear action consequences.

### 2. Taste gate is not optional

The video states that AI still needs a human creative director. This matches our Caster role.

Caster adaptation: generated outputs stay in `candidate_asset` state until Caster selects, rejects or asks for a new generation with a reason.

Anton risk: taste-only selection can overfit atmosphere while under-serving user task, offer clarity, trust and CTA.

### 3. Three-stage production workflow

The useful structure is:

```text
input → create → output
```

For Caster sites this becomes:

```text
production pack → generation/build streams → assembled interface proof
```

### 4. Do not fully automate final assembly in max-quality mode

The output section says that when maximizing quality, final stitching/assembly should not be blindly automated because tools are not creative enough yet.

Caster adaptation: for high-end hero/video-led sites, final crop, sequence, timing and composition require a manual Caster gate even if assets were generated by an agent.

Anton risk: automated assembly can produce a visually plausible page with broken user route or fake proof.

### 5. Production ledger beats chat memory

The visible workflow uses tables/boards for images, prompts, scenes and outputs. That is directly useful.

Caster adaptation: create a prompt/source/output ledger for each generated visual project. Store rejected options and reasons, not just the chosen asset.

## What not to import

- Do not copy the Gentle Monster concept, frames, copy, character or visual rhythm.
- Do not use the video's model/tool pricing as current cost proof.
- Do not treat n8n or any one tool as mandatory for Caster.
